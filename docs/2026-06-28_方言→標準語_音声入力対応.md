# TransHougen 開発まとめ — 方言→標準語の音声入力＆入力UI非対称化

> 対象日: 2026-06-28 / リポジトリ: github.com/kazcchi/transhougen-rn / ブランチ: main（直push運用）
> 最新コミット: `caa22f9f`

---

## 1. 背景・目的

双方向（標準語⇄方言）変換アプリだが、**変換方向で求められる入力が本質的に違う**ことに着目し、入力UIを非対称化した。

- **方言→標準語**：ユーザーは自分の方言名を正確に選べない／選択肢に該当が無いことがある。また方言は口語で「テキスト化しづらい」。
- **標準語→方言**：変換先は一意に決める必要があるので、選択は必須でよい。

---

## 2. 採用した設計

| 方向 | 方言/地方の選択 | 入力方法 |
| --- | --- | --- |
| **方言 → 標準語**（既定） | 不要。任意で**粗い8地方ヒント**（北海道/東北/関東/中部/近畿/中国/四国/九州、既定「おまかせ」=自動判別） | テキスト＋**音声入力** |
| **標準語 → 方言** | チップから変換先を**必須選択**（主要16方言＋番外編4文体） | テキスト |

### 地方ヒントの狙い
同じ語でも地方で意味が異なる「同綴異義語」を補正する。
- 例：「えらい」＝ しんどい（近畿・中部）／ 偉い
- 例：「なおす」＝ 片付ける（西日本）／ 修理する

### 音声入力の方針（重要）
**文字起こしと変換を意図的に分離**した。

```
[🎤] → 録音 → Functions(transcribeAudio) → Whisperで文字起こし
     → 入力欄に追記 → ユーザーが誤認識を修正 → [変換する] → 標準語
```

理由：音声認識（STT）は標準語ベースで学習されており、**強い方言は誤認識しやすい**。一発変換にせず「直してから変換」できるUXで品質を担保。

---

## 3. 変更内容

### フロント（App.tsx）
- 入力UIを方向で**非対称化**（上記）。
- 方言→標準語に **🎤音声入力ボタン**（録音=`expo-audio` の `useAudioRecorder` / HIGH_QUALITY）。
- デフォルト変換方向を **方言→標準語** に変更。
- 方向セグメントの並びを **左＝方言→標準語／右＝標準語→方言** に入替。
- 選択中セグメントを**アクセントカラー（インディゴ）背景＋白文字**で強調。
- **クリアを枠付きボタン化**（入力・結果・エラー表示時に出る／一括クリア）。
- 変換完了・エラー時に**結果カードへ自動スクロール**。

### バックエンド（functions/index.js）
- 新エンドポイント **`transcribeAudio`**（us-central1, 2nd gen, Node.js 22）
  - `multipart/form-data` の `audio` を **busboy** で受信。
  - 上限 **2MB（≒30秒）** 超は 413 で拒否。
  - OpenAI **`gpt-4o-mini-transcribe`**（`language: "ja"`）で文字起こし → `{ text }` を返却。
- 既存 **`dialectConverter`** を更新
  - 方言→標準語プロンプトを**任意の地方ヒント `region` 対応**に（無指定なら自動判別）。
  - レート制限ストアを `transcribeAudio` と**共用**。

### 設定（app.json）
- `expo-audio` プラグイン＋**マイク権限の説明文**（`microphonePermission`）。

## 5. ランニングコスト

**OpenAI $5 ハード上限＋アラートは維持**＝最終防壁。今回の増分はWhisperのみで軽微。

| 処理 | モデル | 概算/回 |
| --- | --- | --- |
| テキスト変換 | gpt-4o-mini | < $0.001 |
| 音声文字起こし（30秒） | gpt-4o-mini-transcribe | $0.001〜0.002 |
| 音声フル（文字起こし＋変換） | — | **≒ $0.003** |

→ $5上限で **音声フル約1,500〜2,500回／テキストのみ5,000回以上**。
（単価は概算）

**多層防御**：$5上限／入力300字・出力500トークン上限／音声2MB(30秒)上限／レート制限30回/時/IP（メモリ内・ベストエフォート）。

⚠️ 注意：レート制限は緩いので「お金の最終防壁」は $5上限。上限到達時はアプリがエラーで止まる（=コスト問題でなく可用性問題）。公開時は **App Check** が効く。
