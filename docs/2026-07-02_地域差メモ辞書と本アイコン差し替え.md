## 2. 「育つ方言辞書」構想の精査で決めたこと

外部で作成した開発方針ドキュメントをレビューし、以下の問題を修正してから実装した。

- **トリガーの誤検出**：「直す」「片付ける」「からって」等の汎用語トリガーは標準語と衝突し、通常文でもメモが出てしまう → 連語・語幹ベースに絞り込み
- **地域体系の不一致**：「関西」「東海」「愛知」等はアプリのUIに存在しない → **正規タクソノミー（9地方＋16方言）に統一**、都道府県は廃止
- **変換方向の非対称**：辞書が方言→標準語の一方向のみ → `dialectPatterns` / `standardPatterns` を分離して**双方向対応**
- **辞書の置き場所**：クライアントにバンドルすると辞書更新のたびに App Store 審査が必要 → **Functions 側に一元化**（クライアントは表示のみ）
- **プロンプト費**：全エントリをプロンプトに常時注入するとスケールしない → **ヒットした分だけ動的注入**
- フェーズ2（ユーザー投稿）は **App Check 実装後**に回す（未認証の書き込みAPIを増やさない）。連絡先メール収集はプライバシーポリシー＋App Privacy 申告の更新が必須になる点に注意

## 3. 地域差メモ辞書（フェーズ1）の実装内容

### 仕組み（辞書はサーバー側に一元化）

```
入力文 + 方向 + 選択地域
  → Functions: detectRegionalUsage() で辞書照合（最大2件）
  → ヒット分だけシステムプロンプトに注入（意味の最終判断は LLM に委譲、機械置換はしない）
  → レスポンスに regionalNotes を同梱
  → App.tsx: 該当時のみ「💡 地域メモ」カードを表示（旧クライアントは無視するので互換）
```

### 初期データ（10エントリ）

| 表現 | 意味 | 地域 | おまかせ時照合 |
| --- | --- | --- | --- |
| 手袋を履く | 手袋をはめる | 北海道・東北 | ○ |
| ゴミを投げる | ゴミを捨てる | 北海道・東北 | ○ |
| 体がこわい | 疲れた・だるい | 北海道・東北 | ○ |
| かたす | 片付ける | 関東 | ○ |
| 鍵をかう | 鍵をかける | 中部 | ×（「買う」と衝突） |
| 机をつる | 机を運ぶ | 中部 | ○ |
| えらい | 疲れた・しんどい | 中部・北陸 | ×（「偉い」と衝突） |
| なおす | 片付ける・しまう | 近畿・中国・四国・九州 | ×（「直す」と衝突） |
| ほかす | 捨てる | 近畿 | ○ |
| からう | 背負う | 九州 | ○ |

### 設計原則（今後エントリを追加するときのルール）

1. `regions` / `dialects` は**9地方・16方言の正規名のみ**を使う（テストで強制）
2. パターンは**語幹・連語ベース**（例:「手袋を履」は 履く/履いて/履いた に一致）
3. 標準語と衝突する語は `matchInAuto: false` にして、**地方を明示選択したときだけ照合**
4. 誤検出しやすい文（「だからって」「そのほかすべて」「誤字を直して」等）は**テストで固定**

- 実装: `functions/regionalUsage.js`（辞書＋検出）、`functions/index.js`（注入＋返却）、`App.tsx`（メモカード＋免責注意文）
- テスト: `cd functions && node --test *.test.js` → **24件パス**（誤検出防止・辞書整合性チェック含む）

### 本番動作確認（curl）

- 「寒いから手袋を履いていきなさい」＋北海道 → 「手袋をはめていきなさい」＋メモ表示 ✅
- 「机の上のものなおしといて」＋近畿 → 「片付けておいて」と正しく解釈 ✅
- 標準語→北海道弁「手袋をはめて、ゴミを捨てておいて」→ **「手袋を履いて、ゴミを投げといてくれよ」**（地域表現を実際に使用）✅
- 通常文「今日はとても疲れました」→ メモなし ✅

### UI追加

- 変換結果の下に「💡 地域メモ」カード（該当時のみ・左に紫アクセント線・控えめな配色）
- 画面下部に免責の注意文（地域差・世代差・文脈による違いの断り書き）→ App Store 審査対策も兼ねる
- 文言変更：地方ヒントのラベルを「地域を選ぶと、より自然に変換できます」に

## 4. アプリアイコンの差し替え

元デザイン `docs/design/TransHougen.png`（紺 `#0f1025` 背景・白T・紫矢印・吹き出し）から生成：

| ファイル | 仕様 | ポイント |
| --- | --- | --- |
| icon.png | 1024²・アルファ無 | App Store 要件。iOS が角丸マスクするので全面使用 |
| adaptive-icon.png | 1024²・ロゴ640px合成 | Android は中央約66%しか表示されないため**セーフゾーン内に縮小**（そのままだと吹き出しが切れる） |
| splash.png | 1284×2778・ロゴ中央 | ソース背景のノイズを単色化してから合成（継ぎ目除去） |
| favicon.png | 48² | Web 用 |

- splash / adaptiveIcon の `backgroundColor` を旧紫 `#4F46E5` → `#0f1025` に統一
- 元デザインは `docs/design/` へ移動（`assets/` に置くと約1.1MBがアプリ本体に同梱されるため）
- 再生成手順：PIL で背景色との色距離≤16の画素を単色置換 → 中央90%クロップ → 各サイズへ
